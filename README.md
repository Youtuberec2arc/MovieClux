# CrunchyEpisode

Anime streaming site — static frontend + a small Vercel serverless backend
that keeps real video URLs off the client entirely.

Live target: `https://crunchyepisode.vercel.app`

## What changed from the old site (summary)

- **Renamed** Anicraze → **CrunchyEpisode** everywhere (titles, footer, page
  titles). Favicon set to `/crunchyepisode.png` (placeholder included —
  swap it for your real logo mark).
- **`watch.html?id=...` removed.** Every episode is now its own static file
  under `/WatchOnline/`, e.g. `WatchOnline/chainsmoker-cat-1x1.html`,
  generated by `scripts/build-watch-pages.js` from `anime-data.js`.
- **Sign In button removed** from the homepage nav and the watch page nav.
- **Sub / MegaPlay server-switch buttons removed** from the watch page —
  each generated page just plays its one configured source. Prev/Next
  episode buttons are kept.
- **"Upcoming" row removed** from the homepage and from `generate-pages.js`.
- **Search fixed** — the homepage search box now filters *every* anime in
  `ANIME_DATABASE` (title, genres, tags), not just what happened to be in
  the visible rows, and results render live as you type. Nav also shows
  live **Series** / **Movies** counts (based on each title's `type`) that
  double as filters when clicked.
- **Embed preview bug fixed** — the old code always tried to load every
  source through the Bitmovin HLS player first and only fell back to an
  `<iframe>` on failure, which is why third-party embed links (pages, not
  raw `.m3u8`/`.mp4` files) often showed a blank/broken preview. Each
  episode now declares its `embedType` (`"iframe"` or `"stream"`) up front,
  so embed-type sources go straight into a plain `<iframe>` with no
  HLS-guessing step in between.
- **Live view counters** are still Firebase-backed and still update in
  real time, but are now keyed **per episode** (`watch/{episodeId}/views`)
  instead of per-anime, and there is no UI anywhere to edit the number —
  it only ever increments via the transaction on page load.
- **Video URLs are no longer in any HTML/CSS/JS shipped to the browser.**
  See the architecture section below.

## How video URLs stay hidden

1. `anime-data.js` (shipped to browsers) only ever contains a **public
   episode id** like `"chainsmoker-cat-1x1"` — never a real URL.
2. The real URL lives only in `api/_data/videoSources.js`, which is under
   `/api`, so Vercel never serves it as a static file — it's only ever
   read inside the serverless function itself. Better still, point the
   `src`/`download` fields at environment variables (see `.env.example`)
   so the URL isn't even sitting in a source file.
3. A watch page's own JS calls `GET /api/token?id=<episodeId>` on load to
   get a token that's cryptographically signed and expires in 5 minutes.
4. It then loads the player from `GET /api/play?id=...&token=...`:
   - `embedType: "iframe"` → the endpoint verifies the token and
     `302`-redirects straight to the real embed page. The real URL never
     appears in your site's HTML/JS — only briefly as a redirect target.
   - `embedType: "stream"` → the endpoint fetches the real file itself
     and **pipes the bytes back**, forwarding `Range` headers so seeking
     still works. The origin CDN URL never appears anywhere in the
     browser, including the Network tab.
5. `/api/play` and `/api/token` both check `Origin`/`Referer` against
   `ALLOWED_ORIGINS` and apply a basic per-IP rate limit (see
   `api/_lib/security.js` — swap in Upstash Redis for real distributed
   limiting at scale, a snippet is included at the bottom of that file).

## Adding a new episode (without ever exposing its URL)

1. Add the episode's **public** metadata to `public/anime-data.js`:
   ```js
   episodes: [
     { number: 2, title: "Episode 2", id: "chainsmoker-cat-1x2", embedType: "iframe", seedViews: 0 }
   ]
   ```
2. Add the matching **private** source to `api/_data/videoSources.js`
   (or, better, just set an env var and reference it there):
   ```js
   "chainsmoker-cat-1x2": {
     type: "iframe",
     src: process.env.VIDEO_SRC_CHAINSMOKER_CAT_1X2,
     download: process.env.DOWNLOAD_SRC_CHAINSMOKER_CAT_1X2
   }
   ```
3. Set `VIDEO_SRC_CHAINSMOKER_CAT_1X2` (and the download one, if used) in
   Vercel → Settings → Environment Variables.
4. Run `npm run build` (or just deploy — Vercel runs it automatically) to
   regenerate `public/WatchOnline/chainsmoker-cat-1x2.html`.

No real URL is ever typed into a file that reaches `git` history in a
public repo, as long as you use the env-var form.

## Local development

```bash
npm install -g vercel        # once
npm run build                # generates WatchOnline/*.html from anime-data.js
vercel dev                   # serves /public + runs /api locally
```

Copy `.env.example` to `.env` and fill in real values before running
`vercel dev`, or the API routes will return 404/empty sources.

## Deploying to Vercel

1. Push this folder to a GitHub repo.
2. Import it in Vercel → New Project.
3. Add every variable from `.env.example` under Project → Settings →
   Environment Variables (use real secrets — don't reuse the example
   values).
4. Deploy. `vercel.json`'s `buildCommand` (`npm run build`) regenerates
   `/WatchOnline` on every deploy automatically, and `outputDirectory`
   (`public`) is what actually gets served statically.

## Still on you before going live

- Replace `public/crunchyepisode.png` with your real favicon and add
  `public/assets/logo.png` + poster/banner art (referenced by
  `anime-data.js`, not included here).
- Fill in a real Firebase project's `databaseURL` in
  `scripts/build-watch-pages.js` (search for `YOUR-PROJECT-ID`) and
  re-run the build.
- Set a real `BITMOVIN_PLAYER_KEY` env var if you use any
  `embedType: "stream"` episodes (Bitmovin-based playback).
- Generate a real `VIDEO_SIGNING_SECRET` with `openssl rand -hex 32`.
- Tighten `ALLOWED_ORIGINS` to just your real deployed domain(s).
